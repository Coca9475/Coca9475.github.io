NOTE：

有一个问题，发现 “My New Post” 被发了2遍，在Hexo server所在的git bash窗口也能看到create了2次。

经验证，在hexo new “My New Post” 时，如果按Ctrl+C将hexo server停掉，就不会出现发2次的问题了。
所以，在hexo new文章时，需要stop server。

解决方案：可能的失效

首先需要安装对应的deploy模块，目前Hexo支持以下服务器的一键部署：

n server
1 git
2 heroku
3 rsync
4 openshift

安装命令为：

1 npm install hexo-deployer-git //将git替换为别的名字就可以安装对应模块

然后对_config.yml做如下设置：

n different property
1 deploy:
2 type: git//if not git, refer to official document
3 repo:
4 branch: [branch]
5 message: [message]

解决方案2：

npm install hexo-deployer-git –save

仍然报错。

npm init 之后 貌似好了-。-

总结

在node.js的命令行工具中进行的npm install hexo-deployer-git –save 是不正确的（猜测是因为生成的package.json不同）
应该在git里边执行此操作，注意修改type：git
